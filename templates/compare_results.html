Why baseline drift matters
Consecutive drift (Vn→Vn+1) is an operational signal — it tells you what changed recently. Baseline drift (Vn vs V1) is the model degradation signal — it tells you how far the current population has moved from the data the model was trained on. A feature can look stable version-to-version while being far from the training baseline.

C11b · Feature Stability Index & Drift Velocity {% set unstable_count = namespace(n=0) %} {% for col, fsi in ds.fsi.items() %}{% if fsi.label == 'unstable' %}{% set unstable_count.n = unstable_count.n + 1 %}{% endif %}{% endfor %} {% if unstable_count.n > 0 %}{{ unstable_count.n }} unstable{% endif %}
FSI & Velocity explained
FSI = 1 − (mean PSI / 0.25). FSI ≥ 0.70 = stable across all versions. Below 0.40 = feature is drifting consistently — consider dropping or re-training. Used in Basel model monitoring and SAS Model Manager.
Drift Velocity = PSI slope per version (least-squares). PSI=0.30 over 10 versions is less alarming than PSI=0.30 in 2 versions. Positive = accelerating drift. Negative = recovering.
